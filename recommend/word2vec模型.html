1.基础知识

Word2vec是语言模型中的一种，它是从大量文本预料中以无监督方式学习语义知识的模型，被广泛的应用到自然语言处理中。

word2vec是用来生成词向量的工具，而词向量与语言模型有着密切的关系。

这个模型看起来简单，但是实现起来还是有点麻烦。例如，先来看看模型参数的个数。刚才是考虑一个给定长度为T的任意句子，理论上就有种可能，而每种可能都要计算T个参数，总共需要计算个参数，当然，这里是简单估算，并没有考虑重复参数，但是这个量级还是有点吓人。此外，这些概率计算好后，还得保存下来，因此，存储这些信息也需要很大的内存开销。

此外，这些参数如何计算呢？常见的方法有n-gram模型、决策树、最大熵模型、最大熵马尔可夫模型、条件随机场、神经网络等方法。

 

1.2 n-gram模型

n-gram是一种基于统计语言的算法，它的基本思想是将文本里面的内容按照字节进行大小为N的滑动窗口操作，形成长度是N的字节片段序列。

每个字节片段称为gram，对所有gram的出现频度进行统计，并且按照事前设定好的阈值进行过滤，形成关键gram列表，也就是这个文本的向量特征空间，列表中的每一种gram就是一个特征维度。

该模型基于这样一种假设，第N个词的出现与前面N-1个词相关，而与其它任何词都不相关，整句的概率就是各个词出现概率的乘积。这个概率可以通过直接从预料中统计N个词同时出现次数得到。常用的是二元Bi-Gram和三元的Tri-Gram。

1.3 神经概率语言模型

什么是向量词呢？简单来说就是，对词典D中的任意词w，指定一个固定长度的实值向量, v(w)就称为w的词向量，m为词向量的长度。

  • 第二层：隐藏层

Softmax是一个非常低效的处理方式，需要先计算每个单词的概率，并且还要计算指数，指数在计算机中都是用级数来近似的，计算复杂度很高，最后再做归一化处理。此后很多研究都针对这个问题进行了优化，比如层级softmax、softmax tree。

 

2. Word2Vec的网络结构

Word2vec是轻量级的神经网络，其模型仅仅包括输入层、隐藏层、和输出层，模型根据输入输出的不同，主要包括CBOW和Skip-gram模型。

CBOW的方式是在知道词的上下文的情况下预测当前词

Skip-gram是在知道词的情况下，对词的上下文进行预测

2.1.1 Simple CBOW Model

先从Simple CBOW model（仅输入一个词，输出一个词）框架说起

  • input layer

    输入的x是单词的one-hot representation（考虑一个词表V，里面每个词都有一个变, 那么的one-hot表示就是一个维度为|V|的向量，其中第i个元素值非零，其余元素全部为0，例如：）

  • 输入层到隐藏层

    输入层到隐藏层之间有一个权重矩阵，其中单词的总个数为V，隐藏层的神经元个数为N，隐藏层得到的值是由输入X乘上权重矩阵得到的（细心的人会发现，0-1向量乘上一个矩阵，就相当于选择了权重矩阵的某一行，如图：输入的向量x是[0, 0, 1, 0, 0, 0]，W的转置乘上X就相当于从矩阵中选择了第3行[2, 1, 3]作为隐藏层的值）

  • 隐藏层到输出层

    隐藏层到输出层也有一个权重矩阵 因此输出层向量y的每一值，其实就是隐藏层的向量点乘权重的每一列，比如输出层的第一个数7，就是向量[2, 1, 3]和向量[1, 2, 1]点乘之后的结果。

  • 输出层

    最终的输出要经过softmax函数，将输出向量中的每一个元素归一化到0～1之间的概率，概率最大的就是预测的词。

  • 损失函数

    损失函数是神经网络最终输出向量和y的交叉熵（cross-entropy），最后用随机梯度下降来更新W和

  • 输出词向量

    一般取中心词对应的权重矩阵，隐藏层和输出层之间的权重矩阵

    训练完毕后，输入层的每个单词与矩阵W相乘得到的向量就是我们想要的词向量，这个矩阵也叫look up table，也就是说，任何一个单词的onehot乘以这个矩阵都将得到自己的词向量

     

2.1.2 CBOW Multi-Word Context Model

了解了Simple CBOW model之后，扩展到CBOW就很容易了，只是把单个输入换成多个输入罢了

对比可以发现，和simple CBOW不同之处在于，输入由1个词变成C个词，每个输入到达隐藏层都会经过相同的权重矩阵W，隐藏层h的值变成了多个词乘上权重矩阵之后加和求平均值。

 

2.2 Skip-gram Model

Skip-gram model是通过一个词去预测多个词的概率。输入层和隐藏层的原理和Simple CBOW一样，不同的是隐藏层到输出层，损失函数变成了C个词损失函数的总和，权重矩阵还是共享的。

3. 加速训练方式

一般神经网络语言模型在预测的时候，输出的是预测目标词的概率，也就是说我每一次预测都要基于全部的数据集进行计算，这无疑会带来很大的时间开销。不同于其他神经网络，Word2Vec提出两种加快训练速度的方式，一种是Hierarchical softmax，另一种是Negative Sampling。

3.2 Negative sampling

Negative Sampling（简称NEG），目的是用来提高训练速度并改善所得词向量的质量。NEG是利用相对简单的随机负采样，能大幅度提高性能。

NEG的细节有点复杂，其本质是利用已知的概率密度函数来估计未知的概率密度函数。简单来说，假设未知的概率密度函数为X，已知的概率密度为Y，如果得到了X和Y的关系，那么X也就可以求出来了。

3.2.1 负采样算法简单介绍

在负采样中，对于给定的词w，如何生成它的负采样集合NEG(w)呢？已知一个词w，它的上下文是content(w)，那么词w就是一个正例，其它词就是一个负例。但是负例样本太多了，我们怎么去选去呢？

在语料库C中，各个词出现的频率是不一样的，我们采样的时候要求高频词选中的概率大，低频词选中的概率小，这就是一个带权采样的问题。

设词典D中的每一个词w对应线段的一个长度，任何采样算法都应该保证频次越高的样本越容易被采样出来，基本思路是对于长度为1的线段，根据词语的词频将其公平地分配给每个词语：

  9. word2vec有哪些参数，有没有什么调参的建议？

    • Skip-gram的速度比CBOW慢一点，小数据集中对低频次的效果更好
    • Sub-sampling frequent words可以同时提高算法的速度和精度，sample建议取值
    • Hierarchical softmax对低词频的更友好
    • Negative sampling对高词频更友好
    • 向量维度一般越高越好，但也不是绝对
    • window size， Skip-gram一般10左右，CBOW一般5左右

  10. word2vec有哪些局限性？

    • word2vec只考虑了上下文信息，而忽略了全局信息
    • word2vec只考虑了上下文的共现性，而忽略了彼此之间的顺序

  11. CBOW和Skip-gram的区别是什么？优缺点

    • 训练速度CBOW应该会更快一点

      因为每次会更新 context(w) 的词向量，cbow是一个老师(中心词)教c个学生(context(中心词))，上下文预测中心词。而Skip-gram只更新核心词的词向量，Skip-gram是c个老师(context(中心词))教一个学生(中心词)。中心词预测上下文, 因为是中心词预测周围词（假设C个），所以过一遍文档是更新了C*n次。所以Skip-gram比cbow时间复杂度高一些，训练起来慢，但是呢，更新次数多，也使embedding更精细，更准确，所以效果好，两者的预测时间复杂度分别是 O(V)，O(KV)

    • Skip-gram对低词频效果比CBOW好

      因为是尝试用当前词去预测上下文，当前词是低频词还是高频词没有区别。但是 CBOW 相当于是完形填空，会选择最常见或者说概率最大的词来补全，因此不太会选择低频词。 cbow是用周围词预测中心词，训练过程中其实是在从output的loss学习周围词的信息也就是embedding，但是在中间层是average的，一共预测V次； skip-gram是用中心词预测周围词，对每一个中心词都有K个词作为output，对一个词的预测有K次，所以能够更有效的从context中学习信息，共预测K*V次，参数更新次数多，因此，skip-gram的训练时间更长 鉴于skip-gram学习的词向量更细致，当数据量较少或者语料库中有大量低频词时，使用skip-gram学习比较合适
